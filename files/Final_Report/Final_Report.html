Problem Summary

In the production of computer animation, it usually requires some skilled animators to implement realistic models and the movements of characters. Physically-based modeling provide an possible solution to synthesize complex shapes automatically. The physical laws of the simulation control the behavior of the models. Then animators can use this physics-based control system to guide their models. With these elastically deformable models, animators will no longer spend huge amount of effort to create elastic surfaces such as cloth, plasticine, and skin and their motions. Moreover, the automatically generated motions are more accurate than the result from manual operation since they will always follow the physics laws in real world.

In this project, I start with cloth simulation and fluid simulation, then I try to model soft objects in sphere shape by combining these two kinds of simulation together. Imagine a balloon filled with water and how would it move under the gravity force and how would it react after the collision with walls. After implement this simulation, I apply the same method to simulate its collision with a rigid object or with another soft objects. With these experiments, it is reasonable to believe this method can generate stable and realistic results under physics laws. Finally, I create two demos for the possible application of this method, they will be presented in the later part of this report.

Previous work

There are two parts of previous work that this project is built on, one is cloth simulation and the other is fluid simulation.

  • Cloth simulation

    The motion of cloth can usually be simulated using a mass and spring system where physics properties such as geometric state, mass distribution and outer forces are represented by particles connected with each other with springs. Applying Newton’s second law of motion :

    x¨=f/m

    to each particles. And rewrite this second order ordinary differential equation into two coupled first order equations as:

    v˙=f/mx˙=v

    and solving these differential equations we can get the simulation of cloths based on the position of each particles. However, this method will not guarantee a stable simulation when time step is large or the springs are too stiff. Using implicit numerical integration method to solve these equations can improve this problem and increase the simulation speed. My cloth simulation code is based on the code in Assignment 5 in CSCE 489: Computer Animation taught by Professor Sueda, which uses an implicit method in solving the whole system. I modified the code to generate a sphere shape cloth. The result of a cloth ball falling is shown below:

    The result of simulating a cloth ball falling into the ground

  • Fluid simulation

    Fluid phenomena are prevalent in every day life. Though seemingly simple and ordinary it is hard and complex to simulate these phenomena to a realistic extent. Luckily, we already have the physics laws to describe the motion of incompressible fluid, the Navier–Stokes equations:

    ρv˙=−ρ(∇⋅v)v−∇p+ν∇2v+F∇⋅v=0

    Though more complicated than the differential equations in cloth simulation, they can still be solved using the standard solution as long as we can find the representation for each element in the equation. Smoothed Particle Hydrodynamics(SPH) is an useful method in fluid simulation with some great properties. The motion of the fluid is represented by the motion of particles and the surfaces of fluid are defined by the state of these particles, which means that all of the information about the fluid is stored in the particles themselves and moves when the particles move. The method I use to simulate fluid is the method described in [CBP05] . In this paper, authors use a double density relaxation procedure to produce realistic small-scale behavior of fluid. Incompressibility, particle anti-clustering and elastic, non-linear plastic effects are all achieved. Here are some fluid simulation results.

    The result of fluid simulation

    The advantage of this method is that it is simple but realistic enough method to simulate viscoelasticity with varying rest length springs. Elastic and non-linear plastic effects are obtained by adding springs with varying rest length between particles. The method is robust and stable, and can animate splashing behavior at interactive framerates.

Description of work

To simulate the soft object with the properties of fluid and cloth, I made following modifications to the simulation process of fluid and cloth to combine them together.

  • In Cloth simulation, I treate fluid particles as rigid objects with large radius but with small collision coefficients.
  • In Fluid simulation, I pad environmental fluid particles in the same positions as the cloth particles to keep the simulated fluid stay within the simulated cloth.
  • In Fluid simulation, I use different rest density for the fluid particles and environmental fluid particles to prevent the fluid particles from escaping the "cage" consists of environmental fluid particles.

The details on why to make these modifications and why they would work is demonstrated as follows.

Large radius, Small collision coefficients

An intuitive idea to simulate the interaction of fluid with the cloth is to view the fluid particles as rigid objects and use the same algorithm for handling collision between cloth and rigid objects. I use the most simple way to handle the collision between cloth and fluid particles, which is calculating the distance between each cloth particles and fluid particles and if they are too close to each other, apply an impulse force to separate them. This force can be expressed as:

f=cdn→d=rcloth+rfluid−||xcloth−xfluid||,

where the c is the collision coefficient that controls how large the force should be, d is the distance that two particles "penetrate" with each other and n→ is the direction from one particle point to the other.

The result if we use small radius, large collision coefficients

If the radius of the fluid particles are too small, the surface of the cloth will not be smooth. More importantly, when the radius is small, there is no enough space for the fluid particles generate consistent force to the cloth particle to make the cloth have a noticeable deformation. If we use large collision coefficient to make the cloth particles move noticeably, the simulation will become unstable.

The result if we use large radius, small collision coefficients

The solution to this problem is use the opposite setting: large radius, Small collision coefficients. Even if it is hard to think it is a reasonable idea to give a particle the collision radius 10 or 15 times larger radius than it usually has. However, if the collision coefficient is reduced properly, we will have stable and realistic result as is shown below. The reason why it works is that we can think that fluid particles influence the cloth particles through force fields rather than collisions. If a cloth particle is closer to a particle than its usual position, it will get pushed back by a slight force. However, if this cloth particle is close to multiple fluid particles at the same time, a significant joined force will generate a noticeable change.

Fix the fluid particles and the cloth is pushed away by the force generated with the large radius, small collision coefficients method

Environmental fluid particles

Now the cloth particles can response to the fluid particles correctly, next step is to keep the fluid particles inside the cloth. One common solution for handling collision of fluid particles is adding extra environmental fluid particles to certain region. When updating the information in the fluid particles count the influence from these environmental fluid particles but only update the position of fluid particles that are actually being simulated. In fluid simulation,the density at a particle is approximated by summing weighted contributions from each neighbor. If current density is different from rest density, there will be a pressure to force the particles to stay at rest density :

Pi=k(ρi−ρrest)

Then padding fixed high density environmental particles at certain region will keep the fluid particles in the simulation away from this region.

I modify this method a little by updating the position of environmental fluid particles with the cloth simulation result from last time step. For fluid simulation, these environmental fluid particles is no different from the static ones but this method provide the cloth particles the ability to apply their influence on the fluid particles. Here is an example that fluid particles are squeezed by cloth.

The Fluid particles generated by perform subdivision of the cloth particles every two rows

Sometimes, there is not enough cloth particles to reach the density requirement. I use the mid-point subdivision on cloth particles. Using their position as old vertices and generate new vertices as the position of environmental particles while keeping the old ones. Usually, 1 or 2 times of subdivision will generated enough environmental particles.

Separate rest density

Even with high density environmental fluid particles padded around the cloth, the fluid particles in the simulation will still run out of the cloth, especially when they are under high impact. It is not feasible that adding more environmental fluid particles can solve this problem form the root, let alone this method will slow down the simulation. The core problem is at some situation where the fluid particles are squeezed to the corner of the cloth and these fluid particles may have higher density than the environmental fluid particles. Using the same equation to calculate the pressure will make them move toward the environmental fluid particles which is the opposite of our whish.

My solution to this problem is setting separate rest density for environmental fluid particles and the free fluid particles. When a fluid particle is calculating current pressure, it will use low rest density for the environmental fluid particles its neighbors and high rest density for other free fluid particles. By doing so, the environmental fluid particles will always pushing free fluid particles away and free fluid particles will always attract each other. Even if it will sacrifice some accuracy in fluid simulation, it will generate much more stable results. Analysis of work

From the results above, we can see that the method I use to simulate soft objects by combining cloth and fluid simulation can generate realistic and stable results under certain parameter setting. The advantages of this method include:

  • Highly-customizable. Users can adjust the parameters in cloth simulation such as spring stiffness, the structure of spring connection, number of particles, number of environmental fluid particles, collision coefficient and in fluid simulation such as rest density, radius for neighbors, paramter k in calculating pressure, and parameters in simulating viscosity and elasticity to obtain desirable output.
  • Easy to implement. For a given 3D object, it only takes two steps to make it filled with fluid and can interact with external forces. The first step is to connect the vertices to create the cloth skin. Then put some fluid particles inside it and everything is done.
  • Provide a method to simulate the interaction between fluid and cloth. Some scenario may require interaction between cloth and fluid. However, some 3D modeling software such as blender do not support interaction between different types. The method I use can provide more freedom for artists for creation and this method has the ability to simulate the activities beneath the outer skin.

The disadvantage of my method is also obvious:

  • Too much parameters. There are so many options to adjust, it may take a long time in adjusting parameters before find the best result. One solution is to make parameters that have more influence on the results adjustable.
  • This method will not work when the fluid particles and cloth particles are too close in the initial state because there is no stable states in such start condition. Luckily, the size of models appears on screen is relative to the camera position, so we can always find camera parameters to make the objects look right. However, it is not so easy to incorporate this method with existing system.

Future work

  • Automatically connect vertices with springs to generate cloth skin. Currently, to simulate a new model, the outer skin is generated manually. It will be useful to have a program to iterate through all vertices and add springs to its neighbor particles. Then all input 3D models can be prepared to be simulated as soft objects instantly.
  • Expand this method to complex models. Currently with only one "kernel" this method can only simulate "sphere" like objects. One improvement is to set multiple groups of fluid particles to the medial axis of a model and when the fluid particles are diverting from their original positions apply a dragging force to keep them stay close.
  • Optimize for neighborhood particles searching. Currently, this method uses simple loops to find neighborhood particles. Implement a spatial hashing grid to store the particles inside a space cube can increase the simulation speed.
  • Separate the time step of cloth simulation and fluid simulation. The fluid simulation can tolerate larger step than cloth simulation. It is reasonable to run cloth simulation multiple times then run fluid simulation once to increase frame rate.
